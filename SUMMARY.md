# Summary: Mapper Algorithm Jupyter Notebook for High School Students

## What Was Created

A complete, production-ready educational package for teaching the Mapper algorithm (topological data analysis) to high school students with **zero coding experience required**.

## Notebook Structure (8 Sections)

### Section 0: Setup
- Install the kmapper library with one command

### Section 1: Why Topology Matters
- Motivating example: two datasets, same statistics, different shape
- Key insight: topology reveals connections and branching (not just averages)

### Section 2: The Mapper Algorithm in 4 Steps
- Filter (lens): assign each data point a single number
- Cover: divide into overlapping bins
- Cluster: group nearby points within each bin
- Nerve: connect clusters that share data points
- Includes a visual 2×2 diagram generated by matplotlib

### Section 3: Parameters You Can Change
- **Filter function** (lens): distance_from_center, main_direction, how_isolated
- **Resolution** (n_intervals): 5–30 slices; higher = more detail
- **Overlap percentage** (overlap_pct): 10–70%; the "glue" that connects clusters

### Section 4: Choose Your Data
- Built-in: Iris (150 flowers, 4 features, 3 species) — quick demo
- Built-in: Wisconsin Breast Cancer (569 patients, 30 features, malignant/benign) — real medical data
- Optional: upload your own CSV file

### Section 5: Run Mapper
- Pre-written code (students run, don't write)
- Normalizes data, applies filter, builds cover, clusters, computes graph
- Colors nodes by class label

### Section 6: Visualize the Result
- Interactive D3.js graph rendered inline
- Node size = cluster density
- Node color = average class label
- Students can hover to explore

### Section 7: Guided Exploration
- Five structured tasks to deepen understanding:
  1. Change resolution, observe graph simplification
  2. Reduce overlap, watch graph disconnect
  3. Try different lenses, discover different patterns
  4. Switch to breast cancer data, identify two lobes
  5. Challenge: find best parameters for class separation

### Section 8: What Did We Find?
- Context: the Nicolau et al. 2011 breakthrough (Mapper discovering a new cancer subtype)
- Why Mapper is powerful (preserves connectivity, interpretable, flexible, scalable)
- Pointers to research and further learning

## Design Principles

### ✓ For High School Students with No Coding Background
- **Plain English**: All concepts explained without jargon
- **Analogies**: Students sorting → overlapping bread slices → graph structure
- **No code writing**: Students run pre-written code; only change parameters
- **Visual**: Four-step diagram, interactive graph, color-coded output
- **Hands-on**: Immediate experimentation with parameters

### ✓ Zero Installation Burden
- **Google Colab**: Click link → runs in browser, all libraries pre-installed
- **Binder**: Click link → Docker environment auto-loads with `requirements.txt`
- **Local**: Single `pip install -r requirements.txt`, then `jupyter notebook`

### ✓ Real Medical Data
- **Iris**: Classic, fast, clear 3-cluster structure (learns the basics)
- **Breast Cancer**: 569 real patients, reveals meaningful two-lobe structure
- **Context**: Notebook mentions Nicolau et al. 2011 cancer subtype discovery

### ✓ Tunable Parameters
- **Three dials** to turn: lens, resolution, overlap
- **Plain-English explanations** for each
- **Prediction tasks**: "What will happen if I change this?"
- **Immediate feedback**: See the graph change in seconds

### ✓ Instructional Materials
- **TEACHER_GUIDE.md**: Learning objectives, discussion questions, extension activities, misconceptions
- **DEPLOYMENT_GUIDE.md**: How to share (Colab, Binder, local, Kaggle)
- **SETUP_CHECKLIST.md**: Pre-class preparation

## Key Features

### 1. Interactive Mapper Graph
- D3.js visualization, renders inline in Jupyter
- Hover for details, click nodes, zoom/pan
- Students can explore the data topology interactively

### 2. Multiple Filter Functions (Lenses)
- **Distance from center** (L2 norm): Simple, radial structure
- **Main direction** (PCA): Variance-based, reveals trends
- **How isolated** (kNN distance): Outlier-based, reveals periphery

### 3. Parameter Exploration
Students predict and test:
- "More resolution → more nodes? Why?"
- "Low overlap → disconnected graph? Why?"
- "Different lens → different shape? Why?"

### 4. Real-World Context
- Breast cancer dataset grounds learning in medicine
- Nicolau et al. 2011 paper shows Mapper's impact (discovered cancer subtype)
- Section 8 bridges to professional applications

### 5. Extension Activities (in TEACHER_GUIDE.md)
- Explore other datasets (Wine, Digits, Penguins)
- Predict-then-experiment tasks
- Research projects with real datasets
- Compare Mapper to k-means, hierarchical clustering, t-SNE
- Parameter sensitivity studies

## How Students Use It

### Online (Recommended)

**Google Colab:**
1. Click the link (e.g., bit.ly/mapper-tda)
2. "Open in Colab" button appears
3. Click through → Colab loads the notebook
4. Run cells from top to bottom
5. Edit parameters in Section 3 and 4
6. Re-run Sections 5–6 to see changes
7. Save work to Google Drive (optional)

**Binder:**
1. Click the link
2. Wait 1–2 minutes for environment to build
3. Notebook opens with all libraries pre-installed
4. No `pip install` needed
5. Sessions expire after 6 hours (temporary storage)

### Local (If Pre-Setup)
1. Clone repo: `git clone ...`
2. Install: `pip install -r requirements.txt`
3. Open: `jupyter notebook mapper_intro.ipynb`
4. Run and experiment

- **Scaffolding**: Concepts before code, simple examples before complex ones

### Accessibility
- **Plain language**: No jargon; all terms explained
- **Colorblind-friendly**: Use size + color distinction
- **Mobile-friendly**: Notebook works on tablets/phones
- **Offline alternative**: Binder for students without Google accounts

## Extensibility

Want to add more content? The notebook is designed to be extended:

### Add More Datasets

- Add cells after Section 6 with specific prompts
